# 🔁 FULL PRODUCT CONTEXT (MANDATORY)

This app is a **gamified hackathon team formation platform** with:

* Profile system (default tab)
* Hackathon discovery
* STRICT team formation flow:

  * User clicks hackathon
  * THEN chooses:

    * Create Team
    * Join Team (recommended only)
* NO global team browsing
* Community (Reddit-style)
* Gamification (XP, levels, badges)
* Chat (ONLY team members)
* Notifications
* Skeleton loaders (NO spinners)

---

# 🎯 GOAL

Ensure the app NEVER breaks under any condition.

---

# 🔐 AUTH EDGE CASES

## Case 1: Session expired

* Detect via Supabase auth state listener
* Immediately:
  → clear local state
  → redirect to /login

---

## Case 2: Invalid login

* Catch error
* Show SnackBar:
  "Invalid email or password"

---

## Case 3: OTP failure

* Show inline error:
  "Incorrect OTP, try again"

---

# 🌐 NETWORK FAILURE HANDLING

ALL repository calls MUST:

try {
await call
} catch (e) {
throw CustomException(message)
}

---

# UI RESPONSE

If error:

* Show message
* Show retry button
* DO NOT crash UI

---

# 📭 EMPTY STATE HANDLING (STRICT)

For ALL lists:

If empty:
→ Show EXACT text:

"Nothing here yet. Come back later."

Apply to:

* hackathons
* teams
* posts
* notifications
* chat

---

# 👥 TEAM SYSTEM EDGE CASES

## Case 1: User already in a team

* Disable "Join Team"
* Show label: "Already in a team"

---

## Case 2: Team is full

* Disable join button
* Show: "Team is full"

---

## Case 3: Hackathon ended

* Disable:

  * Create team
  * Join team
* Show: "Hackathon ended"

---

## Case 4: User tries to access team without joining

* Block access
* Redirect to hackathon screen

---

# 💬 CHAT EDGE CASES

## Case 1: No messages

* Show:
  "No messages yet"

---

## Case 2: Connection lost

* Show:
  "Reconnecting..."
* Retry subscription

---

# 🌐 COMMUNITY EDGE CASES

## Case 1: No posts

* Show empty state

---

## Case 2: Voting spam

* Prevent multiple votes by same user

---

# 🔔 NOTIFICATION EDGE CASES

## Case 1: No notifications

* Show empty state

---

## Case 2: Already read

* Mark visually as read (dimmed)

---

# 🔄 STATE FAILURE CASES

If provider fails:

* set error state
* UI shows retry
